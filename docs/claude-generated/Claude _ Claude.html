This is a really ambitious and well-thought-out concept. Here's my assessment:

What's Compelling

The core problem is real and painful. Legacy COBOL systems at financial institutions are notorious for policy drift—where business rules documented in compliance manuals gradually diverge from what the code actually does. This creates audit risk, regulatory headaches, and makes modernization terrifying.

The approach is architecturally sound. Using tree-sitter for parsing, building an ASG, and storing everything in a graph database is the right technical stack for this problem. The MCP server integration is clever—it means LLMs and other tools can query drift programmatically.

Git blame integration is the killer feature. Being able to say "this policy violation was introduced in commit def789 by Bob on Jan 12" is exactly what compliance officers and modernization teams need. It transforms drift from an abstract problem into an actionable finding.

Where It Gets Challenging

COBOL parsing is harder than it looks. COBOL has dozens of dialects (IBM Enterprise COBOL, Micro Focus, GNU COBOL), compiler directives, and copybooks. Tree-sitter-cobol covers the basics, but real-world COBOL often has preprocessor magic that breaks static analysis.

The Policy DSL needs to be much more expressive. Your example rule checks if a threshold comparison exists, but real business rules are often like "payments over $10K require dual approval unless they're to a pre-approved vendor and the account has been active for 90+ days." The DSL needs to handle multi-condition logic, temporal constraints, and stateful checks.

Test generation from policies is research-level hard. Going from "this policy should hold" to "here's a test suite that validates it" requires understanding control flow, data dependencies, and generating meaningful test data. This feels like Phase 7, not Phase 4.

What I'd Prioritize

If I were building this:

  1. Start with one financial institution as a design partner. Get their top 5 policies and make those work perfectly rather than building a generic system.
  2. Focus on drift detection before test generation. The value proposition is "we found 47 policy violations you didn't know about" not "we generated tests."
  3. Make the MCP server the product. If you can expose drift as queryable data that LLMs and dashboards can consume, you've created infrastructure that others can build on.
  4. Consider a hybrid approach. For some policies, manual annotation might be more practical than DSL queries. Let developers mark code with @policy:dual-approval comments and track those.

Bottom Line

This could be extremely valuable for the right customer—think banks with billions in COBOL that are terrified of compliance violations during modernization. The technical architecture is solid. The main risk is scope creep: trying to be a universal policy engine rather than solving the specific, painful problem of "what changed and did it break our rules?"

Would you be building this for a specific institution, or as a general-purpose product?

import subprocess
import json

def get_violation_commit(filepath, line_number):
    # Run git blame
    result = subprocess.run(
        ['git', 'blame', '-L', f'{line_number},{line_number}', 
         '--porcelain', filepath],
        capture_output=True,
        text=True
    )
    
    # Parse porcelain format
    commit_hash = result.stdout.split('\\n')[0].split()[0]
    
    # Get commit details
    commit_info = subprocess.run(
        ['git', 'show', '--format=%an|%ae|%ad|%s', '-s', commit_hash],
        capture_output=True,
        text=True
    )
    
    author, email, date, message = commit_info.stdout.strip().split('|')
    
    return {
        'hash': commit_hash,
        'author': author,
        'date': date,
        'message': message,
        'file': filepath,
        'line': line_number
    }

Tree-Sitter Crash Course (The Parts You Actually Need)

Tree-sitter is a parser generator that creates concrete syntax trees (CST) from source code. Think of it as a super-fast, incremental parser that understands programming language grammar.

Core Concepts in 5 Minutes

1. The Grammar Someone already wrote the COBOL grammar. You just use it.

2. The Parser You feed it COBOL source, it gives you a tree of nodes.

3. The Query System You write patterns (like regex, but for syntax trees) to find specific code structures.

Day 2: Navigate the Tree (2-4 hours)

Understanding the Tree Structure

Tree-sitter creates a tree where every node has:

  • type - what kind of syntax element (e.g., "if_statement")
  • children - child nodes
  • start_point - (row, column) in source
  • end_point - where it ends
  • text - the actual source code bytes

Walking the Tree (explore_tree.py)

def find_if_statements(node):
    """Find all IF statements in the tree"""
    if_statements = []
    
    if node.type == 'if_statement':
        if_statements.append(node)
    
    for child in node.children:
        if_statements.extend(find_if_statements(child))
    
    return if_statements

def find_comparisons(node):
    """Find all comparison operations"""
    comparisons = []
    
    if node.type in ['comparison', 'relational_expression']:
        comparisons.append(node)
    
    for child in node.children:
        comparisons.extend(find_comparisons(child))
    
    return comparisons

Day 3: Build the Violation Detector (4-6 hours)

The Actual Detection Logic (detect_violations.py)

python
from tree_sitter import Parser
import tree_sitter_cobol as ts_cobol
from dataclasses import dataclass
from typing import List

@dataclass
class Violation:
    file: str
    line: int
    column: int
    policy: str
    description: str
    code_snippet: str
    severity: str

def create_parser():
    parser = Parser()
    parser.set_language(ts_cobol.language())
    return parser

def extract_text(node):
    """Get source code text from node"""
    return node.text.decode('utf8')

def find_violations_in_node(node, filepath, source_lines):
    """
    Find dual-approval violations
    
    Pattern: IF AMOUNT > 10000 ... but no check for second approver
    """
    violations = []
    
    if node.type == 'if_statement':
        # Check if this IF statement has a comparison
        condition = find_child_by_type(node, 'condition')
        if condition:
            comparison = find_child_by_type(condition, 'comparison')
            if comparison:
                # Check if it's comparing an amount to a threshold
                left = find_child_by_type(comparison, 'identifier')
                right = find_child_by_type(comparison, 'number_literal')
                
                if left and right:
                    left_text = extract_text(left).strip()
                    right_text = extract_text(right).strip()
                    
                    # Look for amount variables > 10000
                    if 'AMOUNT' in left_text.upper() and '10000' in right_text:
                        # Check if there's a second approver check
                        if_body = get_if_body(node)
                        has_dual_approval = check_for_dual_approval(if_body)
                        
                        if not has_dual_approval:
                            line_num = node.start_point[0] + 1
                            snippet = get_code_snippet(source_lines, line_num, 5)
                            
                            violations.append(Violation(
                                file=filepath,
                                line=line_num,
                                column=node.start_point[1],
                                policy="Dual Approval Required",
                                description=f"Payment > $10,000 at line {line_num} does not verify second approver",
                                code_snippet=snippet,
                                severity="HIGH"
                            ))
    
    # Recurse through children
    for child in node.children:
        violations.extend(find_violations_in_node(child, filepath, source_lines))
    
    return violations

def find_child_by_type(node, node_type):
    """Find first child with given type"""
    for child in node.children:
        if child.type == node_type:
            return child
    return None

def get_if_body(if_node):
    """Get the body of an IF statement"""
    # This is simplified - real COBOL IF statements can be complex
    for child in if_node.children:
        if child.type in ['statement', 'sentence', 'paragraph']:
            return child
    return None

def check_for_dual_approval(body_node):
    """Check if code verifies a second approver"""
    if not body_node:
        return False
    
    text = extract_text(body_node).upper()
    
    # Look for evidence of second approver check
    # This is a simplified heuristic
    approval_keywords = ['APPROVER-2', 'APPROVER2', 'SECOND-APPROVER', 
                         'DUAL-APPROVAL', 'TWO-APPROVER']
    
    return any(keyword in text for keyword in approval_keywords)

def get_code_snippet(source_lines, center_line, context=3):
    """Get code snippet with context lines"""
    start = max(0, center_line - context - 1)
    end = min(len(source_lines), center_line + context)
    
    snippet_lines = []
    for i in range(start, end):
        marker = ">>> " if i == center_line - 1 else "    "
        snippet_lines.append(f"{marker}{i+1:4d} | {source_lines[i].rstrip()}")
    
    return "\\n".join(snippet_lines)

def analyze_file(filepath):
    """Analyze a COBOL file for violations"""
    with open(filepath, 'r') as f:
        source = f.read()
        source_lines = source.split('\\n')
    
    parser = create_parser()
    tree = parser.parse(bytes(source, "utf8"))
    
    violations = find_violations_in_node(tree.root_node, filepath, source_lines)
    
    return violations

# Test it
if __name__ == '__main__':
    import sys
    
    if len(sys.argv) < 2:
        print("Usage: python detect_violations.py <cobol_file>")
        sys.exit(1)

    for v in violations:
        print(f"📍 {v.file}:{v.line}")
        print(f"   Policy: {v.policy}")
        print(f"   {v.description}")
        print(f"\\n{v.code_snippet}\\n")
        print("-" * 80)

Day 4: Add Git Blame (3-4 hours)

Git Integration (git_trace.py)

python
import subprocess
import re
from dataclasses import dataclass
from datetime import datetime

@dataclass
class CommitInfo:
    hash: str
    author: str
    email: str
    date: datetime
    message: str
    
def get_commit_for_line(filepath, line_number):
    """Use git blame to find the commit that last modified a line"""
    try:
        # Run git blame in porcelain format (easier to parse)
        result = subprocess.run(
            ['git', 'blame', '-L', f'{line_number},{line_number}', 
             '--porcelain', filepath],
            capture_output=True,
            text=True,
            check=True
        )
        
        # Parse porcelain output
        lines = result.stdout.split('\\n')
        commit_hash = lines[0].split()[0]
        
        # Extract author, date, message
        author = None
        email = None
        date = None
        message = None
        
        for line in lines:
            if line.startswith('author '):
                author = line[7:]
            elif line.startswith('author-mail '):
                email = line[12:].strip('<>')
            elif line.startswith('author-time '):
                timestamp = int(line[12:])
                date = datetime.fromtimestamp(timestamp)
            elif line.startswith('summary '):
                message = line[8:]
        
        return CommitInfo(
            hash=commit_hash,
            author=author,
            email=email,
            date=date,
            message=message
        )
        
    except subprocess.CalledProcessError:
        return None

def enrich_violations_with_commits(violations):
    """Add commit info to each violation"""
    enriched = []
    
    for v in violations:
        commit = get_commit_for_line(v.file, v.line)
        
        enriched.append({
            'violation': v,
            'commit': commit
        })
    
    return enriched

# Test it
if __name__ == '__main__':
    # First, make sure you're in a git repo
    import os
    if not os.path.exists('.git'):
        print("Error: Not in a git repository")
        exit(1)

    if commit:
        print(f"Commit: {commit.hash}")
        print(f"Author: {commit.author} <{commit.email}>")

#!/usr/bin/env python3
"""
Stacktalk Demo - Policy Drift Detection
"""

from rich.console import Console
from rich.table import Table
from rich.panel import Panel
from rich.syntax import Syntax
from rich.progress import track
from detect_violations import analyze_file
from git_trace import enrich_violations_with_commits
import sys

console = Console()

def main():
    if len(sys.argv) < 2:
        console.print("[red]Usage: python demo.py <cobol_file>[/red]")
        sys.exit(1)
    
    filepath = sys.argv[1]
    
    # Header
    console.print(Panel.fit(
        "[bold cyan]Stacktalk Policy Drift Detection[/bold cyan]\\n"
        "Connecting code, policies, and commits",
        border_style="cyan"
    ))
    
    # Step 1: Parse
    console.print("\\n[yellow]📖 Parsing COBOL source...[/yellow]")
    violations = analyze_file(filepath)
    console.print(f"[green]✓[/green] Found {len(violations)} policy violations\\n")
    
    # Step 2: Trace commits
    console.print("[yellow]🔍 Tracing violations to commits...[/yellow]")
    enriched = enrich_violations_with_commits(violations)
    console.print(f"[green]✓[/green] Traced {len([e for e in enriched if e['commit']])} violations\\n")
    
    # Step 3: Display results
    if not enriched:
        console.print("[green]No violations found![/green]")
        return
    
    for idx, item in enumerate(enriched, 1):
        v = item['violation']
        commit = item['commit']
        
        console.print(f"\\n[bold red]Violation #{idx}: {v.policy}[/bold red]")
        console.print(f"[dim]{v.description}[/dim]\\n")
        
        # Show code
        console.print(Syntax(
            v.code_snippet,
            "cobol",
            theme="monokai",
            line_numbers=False
        ))
        
        # Show commit
        if commit:
            table = Table(show_header=False, box=None)
            table.add_row("[cyan]Commit:[/cyan]", commit.hash[:8])
            table.add_row("[cyan]Author:[/cyan]", f"{commit.author}")
            table.add_row("[cyan]Date:[/cyan]", commit.date.strftime("%Y-%m-%d %H:%M"))
            table.add_row("[cyan]Message:[/cyan]", f'"{commit.message}"')
            
            console.print(Panel(table, title="[yellow]Origin[/yellow]", border_style="yellow"))
        
        console.print("\\n" + "─" * 80)
    
    # Summary
    console.print(f"\\n[bold]Summary:[/bold]")
    console.print(f"  Files analyzed: 1")
    console.print(f"  Violations: {len(violations)}")
    console.print(f"  Severity: HIGH")

if __name__ == '__main__':
    main()

# Print the ENTIRE tree to understand structure
def print_tree(node, indent=0, max_depth=10):
    if indent > max_depth:
        return
    
    text_preview = node.text.decode()[:30].replace('\\n', '\\\\n')
    print("  " * indent + f"{node.type}: '{text_preview}...'")
    
    for child in node.children:
        print_tree(child, indent + 1, max_depth)
